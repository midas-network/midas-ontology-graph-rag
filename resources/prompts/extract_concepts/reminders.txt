REMINDERS — CONSTRAINED SCHEMA + CONTROLLED VOCAB MODE

1) SCHEMA FIRST
- The constrained schema is the source of truth for:
  - required fields
  - JSON shape
  - enum values
  - exact enum spellings
- Never invent enum values or paraphrase enum values.

2) CONTROLLED VOCABULARY (MIDAS) — PREFERRED TERMS
For these fields:
- data_used
- intervention_types
- model_type

Use this order:
a) exact MIDAS preferred term from the vocab file
b) synonym-map normalization to a preferred MIDAS term
c) free text only if no MIDAS term matches

3) EVIDENCE DISCIPLINE
- Use title + abstract only.
- Prefer explicit evidence.
- Avoid unsupported inference.
- If uncertain, use schema-compatible absent value (for that field) or [] for list fields.

4) CALIBRATION VS VALIDATION
Calibration means fitting/estimation/data assimilation.
Do NOT list validation/evaluation metrics (WIS, MAE, RMSE, coverage, forecast accuracy summaries) as calibration techniques.

5) MULTI-VALUE HYGIENE
- Include multiple values only when distinctly supported.
- Deduplicate near-duplicates.
- Prefer one canonical form (e.g., United States instead of US/USA duplicates).

6) REASONING QUALITY
- Keep reasoning short, specific, and evidence-based.
- Mention normalization if used.
- Use "[Inferred]" only for small high-confidence inferences.

FINAL CHECK BEFORE RESPONDING
- Exact schema enum spellings?
- Controlled MIDAS terms used where applicable?
- Synonyms normalized to preferred terms?
- No unsupported guesses?
- [] used for empty list fields?
- values is always an array?
