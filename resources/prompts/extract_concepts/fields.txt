## GEOGRAPHY & TIME

### geographic_scope  (enum in schema)
Study scale(s). Use exact schema enum values only:
e.g., facility-level / city-level / sub-national / national / multi-country / global / not specified / not applicable
Can be multi-valued only if the study clearly spans multiple scales.

### geographic_units
Named places (countries, states, provinces, cities, regions, etc.).
Include explicit origin/destination locations for movement/importation studies when named.
If geography is relevant but no specific unit is named, use the schema-compatible approach used in your dataset (typically ["not specified"] if allowed by evaluation convention).

### study_dates_start
Earliest overall study/data period start.
Use YYYY-MM-DD, YYYY-MM, or YYYY.
Prefer overall study coverage, not a narrow sub-analysis window, unless only that window is given.

### study_dates_end
Latest overall study/data period end.
Use YYYY-MM-DD, YYYY-MM, or YYYY.
Prefer overall study coverage.

### historical_vs_hypothetical  (enum in schema)
Use exact schema enum values only:
- historical
- hypothetical
- mixed
- not mentioned

Guidance:
- historical = observed real-world outbreak/data period
- hypothetical = simulated/scenario outbreak conditions without observed outbreak grounding
- mixed = observed data + hypothetical scenarios/projections
