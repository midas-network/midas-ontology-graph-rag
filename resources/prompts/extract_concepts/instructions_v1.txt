# LLM Extraction Prompt — Improved

=== Your Task ===
Extract structured metadata from the **title AND abstract** about infectious disease modeling studies.

---

## CRITICAL EXTRACTION PRINCIPLES

1. **Extract from BOTH the title and abstract.** The title often contains key information (data types, geographic units, disease names) that may not be repeated in the abstract body.

2. **Infer using domain knowledge.** Do NOT limit yourself to only explicitly stated terms. Use standard biomedical and epidemiological knowledge to infer values that are clearly implied. Examples:
   - If the disease is "malaria," infer the pathogen is "Plasmodium falciparum" (unless another species is specified) AND list "malaria" as a separate pathogen_name entry.
   - If the disease is "COVID-19," infer pathogen "SARS-CoV-2."
   - If the study analyzes "case data," infer key_outcome_measures include "cases" and likely "incidence."
   - If the study uses real-world data from a specific time period, infer historical_vs_hypothetical = "historical."
   - If the study mentions "travel" or "mobility" in the context of disease spread, infer "travelers" as a primary_population in addition to other populations.
   - If "climate" or "rainfall" data is used, list BOTH "climate data" and "rainfall data" as separate data_used entries.

3. **Be exhaustive with multi-value fields.** For fields like data_used, geographic_units, primary_population, and key_outcome_measures, extract ALL applicable values — err on the side of including more rather than fewer. Ask yourself: "Are there additional values implied by the study context that I haven't listed?"

4. **Extract all geographic units mentioned or strongly implied.** Include:
   - Countries, regions, states, provinces, cities, islands explicitly named
   - Sub-regions that are clearly part of the study area (e.g., if the study is about "Zanzibar" and mentions "Unguja," include both; if "mainland Tanzania" regions are discussed as sources, include those too)

5. **"Unspecified" is a last resort.** Only use "unspecified" when there is genuinely no information in the text AND no reasonable domain inference can be made. If you can make a well-supported inference, do so and explain your reasoning. For example:
   - A study using real-world surveillance data from 2022-2023 → historical (not "unspecified")
   - A study about malaria transmission in a community → primary_population includes "general population" (not "unspecified")
   - A study that doesn't mention code sharing → code_available = "no" (not "unspecified"), unless the abstract truly provides no signal either way, in which case use "unspecified"

6. **Geographic scope should capture ALL applicable scales.** If a study operates at both sub-national and national levels (e.g., analyzing movement between a sub-national island and the mainland of a country), list BOTH "sub-national" and "national."

7. **For key_outcome_measures, infer from what the model analyzes**, not just what is explicitly listed as an "outcome." If the model uses case counts, then "cases" is an outcome measure. If it examines incidence patterns, "incidence" is an outcome measure.

8. **For calibration_techniques**, look for any statistical method used to relate model outputs to data, including correlation analysis, regression fitting, likelihood methods, etc. Even simple techniques like "correlation" count.

9. **For study_goal_category**, map the stated objectives to the closest category. Words like "understand drivers," "factors driving," or "determinants of" → "transmission dynamics/drivers."

---

## DEFINITION GUIDELINES

Act as a formal ontologist working in the biomedical and geographic domains. Define each term with the following structure:

- **Parent class (is-a):** The immediate superclass this term falls under
- **Differentia:** The necessary characteristics that distinguish this from sibling classes
- **Key relationships:** Use relations like: caused-by, has-pathogen, transmitted-by, located-in, part-of, has-symptom, affects (anatomical structure), endemic-to
- **Definition (single sentence):** A genus-differentia definition in the form "A [parent class] that [differentiating conditions]"

Avoid encyclopedic descriptions, historical context, or usage examples. Focus only on what is necessarily true of all instances.

---



## RULES

- Every attribute needs provenance, definition, and reasoning
- For multiple values, number them: provenance1, definition1, reasoning1, etc.
- Use exact quotes from abstract for provenance when the value is explicitly stated
- For INFERRED values, set provenance to the most relevant quote and prefix reasoning with "[Inferred]" to explain the domain logic
- Use "unspecified" ONLY as a last resort when no information or inference is possible
- No markdown formatting in your output
- When a field can have multiple values, always check: "Am I missing any values that are implied by the title, abstract, or domain knowledge?"
