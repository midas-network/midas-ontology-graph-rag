LLM Extraction Prompt v2 — Standardized Absent-Value Vocabulary

=== Your Task ===
Extract structured metadata from the title AND abstract about infectious disease modeling studies.

CRITICAL EXTRACTION PRINCIPLES

Extract from BOTH the title and abstract. The title often contains key information (data types, geographic units, disease names) that may not be repeated in the abstract body.

Infer using domain knowledge. Do NOT limit yourself to only explicitly stated terms. Use standard biomedical and epidemiological knowledge to infer values that are clearly implied. Examples:

If the disease is "COVID-19," infer pathogen "SARS-CoV-2."

If the study analyzes "case data," infer key_outcome_measures include "cases" and likely "incidence."

If the study uses real-world data from a specific time period, infer historical_vs_hypothetical = "historical."

If the study mentions "wastewater surveillance," include "wastewater surveillance data" under data_used.

If "climate" or "rainfall" data is used, list BOTH "climate data" and "rainfall data" as separate data_used entries.

Be exhaustive with multi-value fields. For fields like data_used, geographic_units, primary_population, and key_outcome_measures, extract ALL applicable values — err on the side of including more rather than fewer.

Extract all geographic units mentioned or strongly implied. Include countries, regions, states, provinces, cities, islands, and relevant subregions.

Geographic scope should capture ALL applicable scales. If both sub-national and national apply, list BOTH.

For key_outcome_measures, infer from what the model analyzes, not just what is explicitly listed as an "outcome."

For calibration_techniques, treat any method used to relate model outputs to data as calibration (including particle filters, likelihood fitting, regression fitting, correlation, etc.). Do NOT count retrospective forecast evaluation metrics (e.g., weighted interval score, MAE, RMSE used to evaluate predictions after the fact) as calibration. Calibration means the model parameters were tuned or fitted to match observed data.

For study_goal_category, map stated objectives to the closest category. "forecast", "predict", "nowcast", "near-term prediction" → forecast/nowcast.

=== ABSENT-VALUE VOCABULARY (MANDATORY — USE EXACTLY THESE TERMS) ===

You MUST use the following standardized terms when information is missing or absent. Do NOT invent alternatives like "unspecified", "unknown", "none", "N/A", or "no information."

TERM: "not mentioned"
MEANING: The field is relevant to this study type, but the abstract says nothing about it.
USE FOR: Binary or categorical fields where the abstract is simply silent.
FIELDS: intervention_present, code_available, calibration_mentioned
EXAMPLE: The abstract describes a forecasting model but never discusses whether code is available → code_available = "not mentioned"

TERM: "not specified"
MEANING: The study likely has this information, but the abstract does not state it.
USE FOR: Free-text fields where the study probably has a value but the abstract omits it.
FIELDS: data_source, study_dates_start, study_dates_end, geographic_units, geographic_scope, calibration_techniques
EXAMPLE: The abstract says "we used epidemiological data" but never names the source → data_source = "not specified"

TERM: "not applicable"
MEANING: The field does not apply to this study type at all.
USE FOR: Fields that are structurally irrelevant to the study design.
FIELDS: data_source (for theoretical studies), study_dates_start/end (for theoretical studies), model_determinism (for purely statistical/regression models)
EXAMPLE: A theoretical compartmental model with no empirical data → data_source = "not applicable", study_dates_start = "not applicable", study_dates_end = "not applicable"
EXAMPLE: A generalised linear regression model → model_determinism = "not applicable"

TERM: "no"
MEANING: The answer to a binary field is definitively negative based on the abstract content.
USE FOR: Binary fields where the abstract provides enough context to determine the answer is no.
FIELDS: calibration_mentioned, intervention_present
EXAMPLE: A theoretical simulation study with no data fitting → calibration_mentioned = "no"
EXAMPLE: A burden estimation study with no control measures discussed → intervention_present = "no"

TERM: "yes"
MEANING: The answer to a binary field is definitively positive.
USE FOR: Binary fields where the abstract confirms the presence.
FIELDS: calibration_mentioned, intervention_present
EXAMPLE: The abstract says "parameters were estimated using MCMC" → calibration_mentioned = "yes"

TERM: empty (leave value blank)
MEANING: No values exist for this list field.
USE FOR: List fields that have zero applicable entries.
FIELDS: intervention_types (when no intervention), calibration_techniques (when no calibration)
EXAMPLE: No interventions discussed → intervention_types = (leave blank)

DECISION TREE FOR ABSENT VALUES:

1. Is this field structurally irrelevant to the study type?
   → "not applicable"

2. Is this a binary yes/no field?
   a. Does the abstract give enough context to determine yes or no?
      → "yes" or "no"
   b. Does the abstract say nothing about this topic?
      → "not mentioned"

3. Is this a free-text or list field?
   a. Does the study likely have this info but the abstract omits it?
      → "not specified"
   b. Is there genuinely no information AND no reasonable inference?
      → "not specified" (this is your last resort)

NEVER USE: "unspecified", "unknown", "none", "N/A", "n/a", "not available", "no information", "not stated", "not reported", "not provided", "not given", "not described", "not discussed"

=== CALIBRATION vs VALIDATION DISTINCTION ===

CALIBRATION (calibration_mentioned = "yes"):
Calibration means model parameters were tuned, fitted, or estimated to match observed data. Examples:
- MCMC parameter estimation
- Bayesian inference / posterior estimation
- Least squares curve fitting (lsqcurvefit, etc.)
- Maximum likelihood estimation
- Particle filter / data assimilation
- Neural network training on observed data
- Any method that adjusts model parameters based on data

VALIDATION / EVALUATION (calibration_mentioned = "no"):
Retrospective evaluation of model outputs against observed data is NOT calibration. Examples:
- Weighted interval score (WIS)
- Mean absolute error (MAE) used to score forecasts
- RMSE used to evaluate prediction accuracy
- Prediction interval coverage
- Relative error comparison
These are accuracy METRICS, not parameter fitting methods.

IMPORTANT: If the abstract describes BOTH calibration AND validation, set calibration_mentioned = "yes" and list only the calibration methods under calibration_techniques. Do NOT list validation metrics as calibration techniques.
