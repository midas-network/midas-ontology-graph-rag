**For all controlled-vocabulary fields, you MUST only select values from the provided option lists and return BOTH the numeric `id` and the display `value`.** If a field cannot be determined, return `null`. Do not fabricate information. Flag uncertain fields with `"confidence": "low"` in `extractionNotes`.

Return your output as a single JSON object matching the schema below.

## Instructions

1. **Read the entire source material carefully** before extracting.
2. **Identify the resource type first** from the Type of Record vocabulary.
3. **For ALL controlled-vocabulary fields**: Return `{"id": <number>, "value": "<string>"}` using ONLY values from the vocabularies above. If the source mentions something not in the vocabulary, select the closest match and note it in `extractionNotes`.
4. **For topics**: Select all applicable paths AND their ancestors. For example, a COVID-19 hospitalization forecasting tool requires:
   - `{"id": 68, "value": "disease"}`, `{"id": 69, "value": "disease - infectious disease"}`, `{"id": 182, "value": "disease - infectious disease - vaccine-preventable Disease"}`, `{"id": 70, "value": "disease - infectious disease - vaccine-preventable Disease - COVID-19"}`, plus the viral respiratory path, health system capacity, modeling > forecasting, pathogen > Viruses > SARS-CoV-2, host, and population count paths.
5. **For creators**: Extract all named individuals/organizations with display order and identifiers.
6. **Descriptions**: `briefDescription` = 2-4 sentence summary; `description` = detailed technical description.
7. **Non-specific flags**: `true` if resource is NOT specific to a particular pathogen/host/disease/location.
8. **Be conservative**: Prefer `null` over guessing. Use `extractionNotes` for uncertainty.
